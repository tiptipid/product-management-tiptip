Structural Issues

CRITICAL implementation-plan.md creates dual source of truth

The implementation plan contains detailed content specs for all pages (tool tables, content summaries, lifecycle stage definitions) that partially duplicate and partially contradict the final pages. It is the single largest source of both redundancy and inconsistency.

Recommendation: Archive to _meta/ folder or strip to process-only instructions, removing content specs that now live in the actual pages.

WARNING 4-layer model defined three times

The same explanatory table appears in Pages 02, 03, and 04 with minor wording variations. Define once in Page 00 or 02, then reference from others.

WARNING Scoring process duplicated with drift

Page 03 has a 7-step process; Page 08 has an 8-step process (adds "add row" as step 1). Make Page 08 canonical and have Page 03 reference it.

WARNING Page 07 recommends tools not in Page 00 registry

Page 07 references ChatGPT, GitHub Copilot, Cursor/Antigravity, Linear AI, v0.dev, Miro AI, Notion AI, and ChatPRD — none appear in the Page 00 approved tools list. This violates the "no shadow tools" rule.

Recommendation: Either expand the Page 00 registry or prune Page 07 to only reference approved tools.

Critical AI Gaps

P0 Data privacy when using AI tools

The playbook recommends feeding support tickets, user data, and internal documents into third-party LLMs without any PII handling guidance. This is a compliance and trust risk.

Add to Page 07: PII scrubbing requirements, approved data categories for AI input, data retention policies, which data must NOT be sent to external models. Reference Anthropic's usage policies and Google's PAIR guidelines.

P0 AI product strategy framework

The playbook treats AI only as internal tooling. For a 2026 startup, not having a framework for building AI-powered features is a strategic blind spot.

Add new section or page: Build/integrate/wrap decision framework, AI feature prioritization considerations (data dependency, model uncertainty), data strategy for AI features, AI-native vs AI-augmented product strategy (e.g., conversational booking, AI trip planning).

Reference: Reforge AI PM curriculum, a16z AI playbooks.

P0 MCP security and prompt injection awareness

MCP servers give AI agents read/write access to Jira, Confluence, and Figma. No security guidance provided. A prompt injection through a Confluence page read by an AI agent is a real attack vector.

Add to Page 07: Read-only vs read-write permission guidance, approval gates for AI-initiated writes, prompt injection awareness for data read from external sources.

P1 AI coding workflows for engineering

Page 07 Stage 5 is a stub for engineering AI usage despite the title "AI-Assisted Development." No coverage of context management (CLAUDE.md, .cursorrules), agentic workflows, AI pair programming, or AI-generated tests.

P1 Evaluation framework for agent skills

No way to measure whether agent skills produce good output over time. Need: sample input/output pairs, scoring rubrics applied to AI outputs, regression testing when prompts change.

P1 Prompt versioning strategy

Prompts in .agent/skills/*/prompts/ have no versioning, changelog, or A/B testing framework. Add version tagging, changelog requirements, and rollback procedures.

P1 LLM cost-performance for product features

Model selection guide is internal-only. No guidance for shipping AI features to users: token economics, latency budgets, model selection for production, cost monitoring.

P1 AI literacy and training plan

Team onboarding is limited to "clone repo and invoke skills." Need AI literacy progression: beginner (use skills) → intermediate (customize prompts) → advanced (build new skills).
